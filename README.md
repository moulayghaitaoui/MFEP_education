# منصة التميز الرقمي - وزارة التكوين والتعليم المهنيين (MFEP)

تطبيق ويب متطور يهدف إلى رقمنة قطاع التكوين والتعليم المهنيين في الجزائر، مصمم بواجهات عصرية واحترافية تلبي تطلعات "الإدارة الرقمية الحديثة".

## 🚀 نظرة عامة
المشروع عبارة عن واجهة أمامية (Frontend Only) مبنية باستخدام **Laravel Blade** و **TailwindCSS**، مع تعزيز التفاعل باستخدام **Alpine.js**. المنصة مصممة لدعم اللغة العربية (RTL) بالكامل وتوفر تجربة مستخدم فريدة لأربعة أدوار رئيسية.

---

## 📄 بيان حالة المشروع (Project Status Statement)

### 1. طبيعة المشروع (Project Nature)
يُعد هذا المشروع نموذجاً أولياً تفاعلياً (**Interactive UI/UX Prototype**) تم تطويره كلياً كواجهة أمامية (Frontend-Only) بهدف تقديم تصور بصري شامل ورؤية استراتيجية لمستقبل التحول الرقمي في قطاع التكوين والتعليم المهنيين. المشروع لا يهدف إلى أداء وظائف برمجية خلفية في مرحلته الحالية، بل يركز على صياغة تجربة مستخدم (**User Experience**) عصرية تليق بمؤسسة حكومية سيادية.

### 2. الأهداف الاستراتيجية (Core Objectives)
*   **التجسيد البصري**: تقديم تصور ملموس لوزارة التكوين والتعليم المهنيين حول كيفية دمج التكنولوجيا في إدارة المعاهد والمديريات.
*   **محاكاة رحلة المستخدم**: تمكين أصحاب القرار من تجربة المنصة من منظور أربعة أدوار رئيسية (مدير النظام، المديرية الولائية، الأستاذ، المتربص).
*   **اختبار تدفق البيانات**: عرض كيفية تنظيم البيانات الضخمة (Big Data) والإحصائيات الوطنية بشكل مبسط واحترافي.

### 3. المواصفات الفنية والتقنية (Technical Context)
*   **هيكل التطوير**: تم بناء النموذج باستخدام إطار العمل **Laravel Blade** مع الاعتماد الكلي على **TailwindCSS** للتنسيق البصري، و **Alpine.js** لإضافة التفاعلية الحركية.
*   **نظام المحاكاة (Simulation Mechanism)**:
    *   **دخول المستخدم**: تتم عملية الدخول عبر واجهة تسجيل دخول (Login UI) تجريبية تتيح الانتقال اليدوي بين الأدوار.
    *   **إدارة الأدوار (Role Management)**: يتم محاكاة صلاحيات الأدوار المختلفة برمجياً على مستوى الواجهة فقط (Frontend Logic)، دون وجود نظام توثيق (Authentication) أو قاعدة بيانات حقيقية.
    *   **البيانات الافتراضية (Dummy Data)**: كافة الجداول، المخططات البيانية، والإشعارات هي بيانات افتراضية تهدف لشرح آليات العرض والتحليل المستقبلي.

### 4. تجربة المستخدم والتصميم (UX/UI Design)
تم اعتماد لغة تصميم **"Digital Government"** التي تجمع بين الطابع الرسمي والاحترافية التقنية، مع استلهام جماليات الأنظمة الأمنية الحديثة (**Security-Grade UI**) في لوحات المراقبة والتحكم، لضمان أعلى مستويات الدقة والوضوح.
*   **التنقل الاستجابي**: النظام مصمم ليكون متوافقاً بالكامل مع كافة الشاشات (Responsive Design).
*   **دعم اللغة العربية**: تم ضبط الواجهات لتدعم الاتجاه من اليمين إلى اليسار (RTL) بشكل أصيل مع استخدام خطوط طباعية حديثة.

### 5. ملاحظة هامة (Technical Disclaimer)
هذا النظام هو **نسخة عرض (Demo Only)** مخصصة لغرض العرض التقني والجمالي أمام اللجنة المختصة بوزارة التكوين والتعليم المهنيين. لا يحتوي النظام على منطق برمجي خلفي (Backend Logic) أو ربط بقواعد بيانات، ويهدف حصراً لاستطلاع الآراء وتثبيت المتطلبات البصرية والوظيفية قبل الانتقال لمرحلة البرمجة الفعلية.

## 🎨 الهوية البصرية والوطنية (National Branding)
تم تصميم المنصة لتجسيد الهوية الرسمية للجمهورية الجزائرية الديمقراطية الشعبية، وتحديداً وزارة التكوين والتعليم المهنيين:
*   **الألوان الرسمية**: الاعتماد الكلي على **الأخضر الجزائري (Algerian Green)** و **الذهبي الملكي (Royal Gold)** المستوحى من ختم الوزارة الرسمي.
*   **الشعار الموحد**: دمج شعار الوزارة في كافة الواجهات مع نظام Fallback يضمن ظهور الهوية حتى في حالات التحميل البطئ.
*   **الخطوط**: استخدام خط **Tajawal** العصري لضمان وضوح النصوص الرسمية والتقنية.

## 👥 الأدوار والوظائف المدعومة بالتفصيل

### 1. الإدارة العليا (Super Admin Portal)
*   **مركز التحكم الوطني (National Command Center)**: لوحة معلومات ذكية تعرض إحصائيات حية لعدد المتربصين، الأساتذة، والمعاهد عبر 58 ولاية.
*   **مراقبة الأداء الإقليمي**: خرائط تفاعلية توضح توزيع المؤسسات التكوينية ومستويات الإنجاز.
*   **نظام التنبيهات السيادي**: مراقبة سجلات الدخول والنشاطات الحساسة لضمان أمن البيانات.

### 2. المديرية الولائية (Regional Directorate Portal)
*   **إدارة الخارطة التكوينية**: متابعة المعاهد (INSFP) ومراكز التكوين (CFPA) داخل الولاية.
*   **لوحة أداء المتربصين**: متابعة نسب النجاح، الحضور، والغياب بشكل إقليمي.
*   **التسيير البيداغوجي الموحد**: الإشراف على الجداول الزمنية (Schedules) وتوزيع الأفواج.

### 3. فضاء الأستاذ (Teacher/Instructor Hub)
*   **منشئ الدروس التفاعلي (Interactive Lesson Builder)**: واجهة متطورة تتيح للأستاذ دمج النصوص، الفيديوهات، والأسئلة داخل محتوى الدرس.
*   **نظام بناء الاختبارات (Advanced Quiz Builder)**: أداة ديناميكية لإنشاء اختبارات (QCM، نصوص، فراغات) مع معاينة فورية.
*   **لوحة تحليل الطالب**: تحليل بياني لمستوى كل متربص ونقاط القوة والضعف لديه.

### 4. فضاء المتربص (Student Learning Experience)
*   **مشغل الدروس الذكي**: مشغل فيديو تفاعلي يتوقف عند نقاط محددة لطرح أسئلة (Inline Questions) لضمان الفهم.
*   **مركز المسارات المهنية**: تتبع التقدم في المسارات التكوينية والحصول على بادجات (Badges) تشجيعية.
*   **المحفظة الرقمية**: عرض وتحميل الشهادات الموقعة رقمياً والتحقق منها عبر رمز QR.

## 🏗️ الهندسة البرمجية (Architectural Fixes)
تم إجراء تحسينات هيكلية جذرية لضمان استقرار العرض:
*   **Layout Decoupling**: تم فصل القوالب (Layouts) لكل دور بشكل مستقل (`layouts.admin`, `layouts.direction`, `layouts.public`) لمنع تداخل عناصر الـ Sidebar والـ Navbar.
*   **Component-Based UI**: بناء الواجهات باستخدام مكونات Blade (Blade Components) لضمان إعادة استخدام الأكواد وسهولة الصيانة.
*   **Sticky UI Mechanisms**: تفعيل أنظمة التثبيت الذكي للقوائم العلوية والجانبية لضمان تجربة تصفح سلسة في الشاشات الطويلة.

## 🛠️ التقنيات المستخدمة
*   **Framework**: Laravel 11.x
*   **Styling**: TailwindCSS 3.x (مع تخصيص لوحة ألوان `gov-green` و `gov-gold`).
*   **Interactivity**: Alpine.js (لإدارة الحالة، الـ Modals، والأسئلة التفاعلية).
*   **Icons**: Lucide Icons (أكثر من 50 أيقونة مخصصة).
*   **Fonts**: Tajawal (Arabic) & Inter (English).

## ⚙️ التشغيل والمعاينة
للتشغيل المحلي، تأكد من وجود Node.js 20+ و PHP 8.2+:

1. تثبيت الاعتمادات:
```bash
composer install
npm install
```

2. بناء الأصول (Vite):
```bash
npm run build
```

3. تشغيل الخادم:
```bash
php artisan serve
```
